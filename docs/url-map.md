## Notas de migración

- **trailingSlash**: `always` (ya configurado en `astro.config.mjs`).
- **Estructura de routing Astro**: las categorías son rutas de primer nivel (`/cirugia-oral/`) y los posts cuelgan de ellas (`/cirugia-oral/{slug}/`). Esto exige rutas dinámicas `[categoria]/[slug].astro` + `[categoria]/index.astro`, cuidando no colisionar con las páginas estáticas (`/seguros-dentales/`, etc.).
- **Colisión potencial**: `/seguro-dental-adeslas-segurcaixa/` es categoría Y tiene un post hijo (`/listado-de-seguros/`). Verificar que el routing dinámico de categorías no pise las páginas estáticas de nombre similar (`/seguros-dentales/` es página, no categoría).
- **Tags y categorías**: generan páginas de archivo (listados). Confirmar con el operario si se migran como listados o si alguna queda como AMPLIACIÓN diferida (nunca eliminándolas del mapa).
